# Offer 与垂类评估手册

更新时间：2026-06-08

本文用于回答：一个 Offer 或垂类是否值得进入 Ads 套利测试。重点不是系统怎么录入，而是投手和运营在投放前如何判断“能不能做、怎么小测、什么时候放弃”。

## 1. 评估原则

不要只看 payout。高 payout 通常伴随：

- 政策风险更高。
- CVR 更低。
- 审核和拒付更严。
- 回款周期更长。
- 追踪和对账更复杂。
- 素材和页面更容易产生夸大承诺。

一个好 Offer 至少满足：

- 用户意图清晰。
- 页面能提供真实信息增量。
- 广告承诺和最终页面一致。
- 收入回传可验证。
- 扣量规则能理解。
- 现金流能撑过回款周期。
- 账号和域名风险可控。

## 2. 评估维度

| 维度 | 问题 | 通过信号 | 风险信号 |
| --- | --- | --- | --- |
| Payout | 单次转化或 RPM 是否足够覆盖 CPC | Payout 稳定、规则清楚 | 高 payout 但规则模糊、频繁拒付 |
| CVR | 用户从点击到收入的概率 | 有历史数据或同垂类参考 | 只靠供应商承诺，无真实样本 |
| CPC | 目标流量成本 | 长尾词或细分国家可控 | 核心词竞争极高 |
| 政策 | 是否受限制或敏感 | 可合规表达，页面透明 | 金融收益、医疗疗效、博彩、加密、成人、仿牌 |
| 页面 | 是否能做出真实内容 | 可做比较、指南、工具、测评 | 只能做跳转页或广告堆叠 |
| 追踪 | 收入能否对上点击 | 支持 SubID/S2S/postback | 只能看汇总报表 |
| 扣量 | 拒付和扣量是否透明 | 有扣量原因、可分来源 | 月底统一扣，无法解释 |
| 回款 | 现金流是否可承受 | 周结或稳定月结 | 长账期、高拒付、预付款压力 |
| 扩量 | 是否能扩大真实流量 | 国家/关键词/素材可扩 | 小样本有效，扩量后 RPV 快速崩 |
| 风控 | 是否容易触发账号和域名风险 | 有清晰政策备注和审计 | 需要 cloaking、补点击、绕审核才跑得动 |

## 3. 评分模型

建议 100 分制：

| 项目 | 权重 |
| --- | --- |
| 单位经济：RPV、CPC、payout、CVR | 25 |
| 政策和账号风险 | 20 |
| 页面内容可做性 | 15 |
| 追踪和对账可验证性 | 15 |
| 扣量和回款风险 | 10 |
| 扩量空间 | 10 |
| 团队经验匹配 | 5 |

建议动作：

- 80 分以上：可小预算测试。
- 65~79：先修页面、追踪或政策表达，再小测。
- 50~64：只做研究，不进投放。
- 50 以下：拒绝。

如果政策风险为 high，不能靠总分抵消，必须先做风险审计。

## 4. 常见垂类判断

### 4.1 B2B SaaS / 工具软件

优点：

- 用户意图清晰。
- 可做比较、指南、清单、ROI 计算器。
- 页面内容容易做出信息增量。
- CPA/CPL 或订阅收入较稳定。

风险：

- 品牌词、竞品词和商标表达要谨慎。
- 不应暗示官方关系或虚假背书。
- 页面不能只是导流到供应商官网。

适合动作：

- Search 小预算长尾词测试。
- 比较型创意。
- 页面版本按行业、规模、用例拆分。

### 4.2 保险 / 金融服务

优点：

- 用户意图强。
- payout 往往较高。
- 国家和细分产品可拆。

风险：

- Google Ads 金融产品和服务政策、当地法规、资质和披露要求可能适用。
- 个性化广告限制可能影响定位。
- 夸大收益、保证审批、误导性价格和隐藏费用风险高。
- Lead 质量审核和拒付可能严格。

适合动作：

- 先做政策和资质核查。
- 页面加入清晰披露、条件、费用、隐私和联系信息。
- 只用可验证的比较和说明，不承诺结果。

### 4.3 医疗 / 健康

优点：

- 搜索意图明确。
- 内容型页面可承接教育需求。

风险：

- 医疗和药品政策限制强。
- 个性化广告对健康相关兴趣和敏感类别限制严格。
- 疗效承诺、诊断建议、处方药、未批准治疗风险高。

适合动作：

- 避免诊断和疗效保证。
- 明确内容用途和专业免责声明。
- 不做敏感健康状态的人群定向。

### 4.4 教育 / 职业培训

优点：

- 可做清单、比较、课程路径。
- CPL 常见，内容空间大。

风险：

- 夸大就业、收入、录取、证书价值会触发误导。
- 地区资质、学校关系和认证声明要真实。

适合动作：

- 做职业路径、课程比较、费用说明。
- 明确数据来源和适用条件。

### 4.5 本地服务

优点：

- 意图强，关键词明确。
- Lead-gen 可按城市拆分。

风险：

- 资质、营业地点、服务范围、价格和响应时间必须真实。
- 某些本地服务可能涉及验证要求。
- 假地址、虚假评价和冒充品牌风险高。

适合动作：

- 城市/服务拆分页面。
- 明确服务商关系、费用和隐私。
- 用真实商家信息和联系方式。

### 4.6 电商 / 比较购物

优点：

- 商品信息可比较。
- Search 和 Shopping 意图清楚。

风险：

- Misrepresentation、价格不一致、虚假折扣、缺少退换货/联系信息风险高。
- Merchant Center 和 Ads 资料不一致会扩大影响。
- 仿牌和商标问题风险高。

适合动作：

- 保持商品、价格、库存、退换货、联系方式一致。
- 避免假倒计时、假评价、虚假授权。

### 4.7 加密 / 投资 / 高收益项目

优点：

- payout 可能高。

风险：

- 政策、资质、欺诈和误导风险极高。
- 容易需要夸大收益或规避审核才有点击。
- 账号、域名、付款资产受损概率高。

建议：

- 默认拒绝，除非有明确资质、合法披露和保守文案。

### 4.8 博彩 / 游戏 / 抽奖

优点：

- 用户付费意图可能强。

风险：

- 赌博和游戏政策、国家限制、牌照要求和年龄限制严格。
- 受众定向和页面披露复杂。

建议：

- 默认 high 风险，需要独立政策审计和资质确认。

## 5. Offer 资料清单

进入测试前，至少收集：

- Offer 名称和垂类。
- 国家、语言、允许设备。
- Payout 模型和金额。
- 合格转化定义。
- 拒付和扣量规则。
- 回传方式和字段。
- 结算周期和付款条件。
- 禁止流量来源。
- 禁止关键词、素材和页面表达。
- 是否允许品牌词、竞品词、预填表单、激励流量。
- 隐私、条款、资质和披露要求。

缺这些资料，不建议进入投放草稿。

## 6. 测试设计

一个 Offer 小测必须写清：

```text
测试目标：
假设：
国家/语言/设备：
关键词或流量来源：
页面版本：
创意角度：
预算：
最小样本：
硬止损：
成功标准：
风险审计状态：
追踪和收入来源：
```

成功标准示例：

- 达到 200 paid clicks。
- arrival rate > 75%。
- RPV_ads_click > CPC * 1.3。
- ROI_reported > 20%。
- 无 high 风险审计未处理。
- 扣量在可解释范围。

## 7. 拒绝标准

满足任一条件应拒绝或暂停：

- 需要 cloaking 才能过审。
- 需要补点击、刷展示或模拟自然流量才能维持报表。
- 需要使用代理/指纹/Worker 规避关联检测。
- 需要封禁后换账号继续跑。
- 页面无法提供真实内容，只能桥接到下游。
- payout 高但拒付规则不透明。
- 无法按 SubID 或 click_id 对账。
- 广告承诺必须夸大收益、疗效、官方关系或价格。

## 8. 系统落地

当前系统可用：

- `/offers` 记录 Offer 和政策备注。
- `/calculators` 做单位经济和机会评分。
- `/risk-audits` 记录政策和高风险能力审计。
- `/sources` 记录政策和来源 URL。
- `/metrics/import` 导入测试结果。
- `/optimization` 查看后续动作。

建议运营流程：

1. 先用本文评估 Offer。
2. 通过后录入 `/offers`。
3. 用 `/calculators` 做测算。
4. 通过 `/risk-audits` 做上线前审计。
5. 小预算测试。
6. 导入指标并复盘。
