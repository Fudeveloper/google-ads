# 高风险能力专题索引

更新时间：2026-06-09

这些专题用于解释 ADXKit 类产品中常被提到、但在合规和账号安全上高度敏感的能力。每份文档都包含原理、行业诉求、风险、平台规则、识别逻辑、合规替代和来源 URL。

本目录的完成标准是：逐点复刻可合规沉淀的行业知识、流程、审计和安全替代能力。换句话说，这些点不是“不复刻”，而是把可交付部分定义为原理解释、行业诉求、平台逻辑、识别方法、审计字段、SOP 和安全替代；对执行型绕过、刷量、会话接管和规避封禁能力，不交付可执行实现。每个专题都要回答：

- 行业为什么会要这个能力。
- 技术本质和广告系统原理是什么。
- 平台、账号安全或无效流量系统为什么会把它视为高风险。
- 正常业务场景和规避场景的边界在哪里。
- 系统中用什么安全能力替代这个诉求。
- 依据了哪些公开来源 URL。

安全边界：

- 不提供绕过步骤。
- 不提供刷量、补点击或流量模拟实现。
- 不提供 Cookie 复用和后台接管实现。
- 不提供 cloaking 分流实现。
- 不提供规避封禁换号流程。

系统落地：

- 用 CSV / Google Ads Scripts JSON / 官方 API 方向替代 Cookie 操作。
- 用人工审核、审计日志和权限管理替代绕过安全挑战。
- 用指标导入和异常分析替代补点击。
- 用真实业务隔离和账号健康复盘替代规避关联。
- 用合规链接计划替代 cloaking。
- 用申诉、修复和复盘替代封禁后换号。
